# 작업 요약 (2025-11-30)

## 1. 버그 수정 (Bug Fixes)

### A. PAT 프로그램 목록 누락 해결
*   **현상**: PAT에 프로그램이 여러 개 있어도 1개만 표시되는 문제.
*   **원인**: `ts_parser_core.py`의 파싱 루프 조건이 부정확했고, `Program 0` (NIT)를 무시하는 로직이 존재했음.
*   **수정**: 
    *   파싱 루프 조건을 `while i + 4 <= limit`로 명확히 하여 데이터 끝까지 읽도록 개선.
    *   `if prog_num != 0` 조건을 삭제하여 NIT를 포함한 모든 프로그램 정보를 수집.

### B. PAT 정보 화면 미표시 해결
*   **현상**: PAT(PID 0)를 선택해도 우측 분석 화면에 "Select a PID"라고 뜨며 정보가 안 나옴.
*   **원인**: GUI 코드에서 `if not selected_pid:` 조건문이 `0`을 `False`로 처리함.
*   **수정**: `if selected_pid is None:`으로 변경하여 PID 0도 정상 선택으로 인식.

## 2. UI/UX 개선 (Improvements)

### A. 직관적인 클릭 동작 (Tree & Toolbar Sync)
*   **PAT 타이틀 클릭**: 
    *   PAT 필터 활성화 (툴바 버튼 동기화).
    *   하위 프로그램 선택 해제 (트리 하이라이트 끔).
    *   즉시 PAT 정보 화면 표시.
*   **Program 클릭**:
    *   해당 프로그램의 **PMT PID를 자동 선택**.
    *   PMT 필터 활성화 및 정보 화면 즉시 표시.
*   **PID(Video/Audio) 클릭**:
    *   **단독 집중 모드**: 툴바의 전역 필터(Video, Audio 버튼)를 모두 끄고, 선택한 PID 하나만 필터링하여 보여줌.

### B. 툴바 버튼 동작 강화
*   **[PAT] 버튼**: 클릭 시 단순히 필터만 켜는 게 아니라, **PID 0을 자동 선택**하고 화면을 이동.
*   **[PMT] 버튼**: 클릭 시 현재(또는 첫 번째) 프로그램의 **PMT PID를 찾아 자동 선택**.

### C. 초기 실행 편의성
*   파일 열기 시 `Program 0` (NIT)을 건너뛰고, **실제 방송 프로그램(Program 1 등)**을 자동으로 선택하여 바로 PMT 정보를 볼 수 있도록 함.
